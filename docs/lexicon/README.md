# KEY Lexicon

The KEY Lexicon is the canonical vocabulary for the project.

It exists so that the manifesto, architecture, prototype, evaluation system, agents, and future codebase use foundational terms consistently.

## Maintenance Principle

When the project introduces a foundational term, it should receive one precise definition here. Other documents should reference or remain consistent with that definition rather than creating competing meanings.
