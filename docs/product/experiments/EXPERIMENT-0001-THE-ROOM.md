# EXPERIMENT-0001 — The Room

## Status

Planned

## Primary Hypothesis

Players will perceive consequences as meaningful when an earlier choice changes later character behavior, relationship state, resource control, and available options in a traceable way.

## Secondary Questions

- Can three agents remain inside distinct knowledge boundaries?
- Can agents act during player absence?
- Can a promise create a future obligation?
- Can withholding information matter differently from lying?
- Can the same initial state produce more than one coherent outcome?

## Environment

One enclosed meeting room during a local crisis.

Three agents dispute control of one scarce resource needed before the end of the simulated hour.

The room is intentionally abstract enough to avoid art and genre work becoming the experiment.

## Required Initial Conditions

### Agent A

- possesses decisive private knowledge
- wants immediate control of the resource
- fears public loss of status
- distrusts Agent B because of a past event

### Agent B

- has a legitimate claim to the resource
- misunderstands Agent A's motive
- depends on Agent C for future support
- values fairness but will compromise under pressure

### Agent C

- can verify part, but not all, of the disputed truth
- wants the conflict contained
- fears responsibility for a bad outcome
- owes one agent a prior favor

## The Resource

The resource must be:

- singular or sharply limited
- immediately useful
- important to more than one agent
- transferable
- capable of producing delayed consequences

Its exact fiction may vary between test fixtures, but its systemic role must remain constant.

## Player Actions

The first interface permits:

- ask a directed question
- reveal known information
- withhold information
- make a promise
- make a false claim
- give the resource
- take the resource
- support an agent's claim
- leave the room
- advance time

## Simulation State

The experiment tracks:

- canonical facts
- each observer's knowledge
- each agent's beliefs
- resource ownership and access
- directional trust
- fear and dependency
- promises and obligations
- remembered events
- current goals
- pressure
- available actions
- perceived possibilities

## Required Causal Trace

Every selected agent action must record:

1. what the agent knew
2. what the agent believed
3. what the agent wanted or feared
4. which relationship or obligation mattered
5. which actions were considered
6. why the selected action dominated
7. what state changed afterward

## Time Advancement

At least one segment must occur after the player leaves or explicitly advances time.

During that interval, agents may:

- speak privately
- transfer or secure the resource
- fulfill or violate a promise
- seek verification
- retaliate
- cooperate
- revise a belief

The system must not select these actions solely to produce a dramatic return. They must emerge from agent state and constraints.

## Test Variants

Run at least four controlled variants:

1. Player gives the resource to Agent A.
2. Player gives the resource to Agent B.
3. Player promises support but leaves without transferring it.
4. Player reveals Agent A's private information before leaving.

Each variant should produce a distinct later state without requiring separate authored endings.

## Success Signals

- A majority of playtesters correctly identify the earlier action that caused the later state.
- Testers describe at least one agent action as intentional rather than random.
- No agent uses inaccessible information.
- Promise, trust, and resource changes persist across time advancement.
- At least two variants close or open different future actions.
- Internal causal traces match the player's perceived explanation closely enough to be legible.

## Failure Signals

- Players believe the result would have happened regardless of their choice.
- Agents act mainly to create drama rather than from modeled causes.
- Knowledge leaks between agents.
- State changes are recorded but not expressed in behavior.
- Outcomes differ only in wording.
- The causal trace requires post hoc explanation unsupported by state.

## Explicit Non-Goals

This experiment does not test:

- rich dialogue quality
- large language model creativity
- visual presentation
- combat
- world generation
- long-form plotting
- complete emotional realism
- adaptive UI
- adaptive game design

## Advancement Criteria

EXPERIMENT-0001 may advance to implementation review only after:

- the state schema is defined
- action provenance is inspectable
- all test variants can run from the same rules
- automated knowledge-boundary checks exist
- human playtest questions and recording format are ready

It may advance to EXPERIMENT-0002 only after results are documented and the primary hypothesis is supported, revised, or rejected.
