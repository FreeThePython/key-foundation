# KEY Memory Engine Specification

## Purpose

The Memory Engine preserves the causal and interpretive continuity that allows events to remain active through time.

## Research Principle

> Events occur once. Their consequences, interpretations, obligations, and memories may continue shaping the future.

## Core Responsibilities

- retain event provenance
- preserve consequence chains
- maintain agent, relationship, and institutional memory
- distinguish canonical records from remembered or contested versions
- support forgetting, distortion, concealment, and rediscovery
- expose relevant history to other engines without collapsing context

## Requirements

### M-001 — Event Provenance

Every significant state change must be traceable to one or more events, processes, or prior conditions.

**Evidence:** CASE-0001 through CASE-0004  
**Related hypotheses:** H-001, H-014  
**Confidence:** Moderate

### M-002 — Consequence Persistence

Consequences remain active until transformed, resolved, credibly forgotten, or superseded.

**Evidence:** CASE-0001, CASE-0003, CASE-0004; contrast from CASE-0002  
**Related hypotheses:** H-001, H-005, H-008  
**Confidence:** Moderate

### M-003 — Multiple Memory Layers

The engine must distinguish canonical event record, individual memory, shared memory, institutional record, and public narrative.

**Evidence:** CASE-0001, CASE-0004  
**Related hypotheses:** H-007, H-012, H-016  
**Confidence:** Moderate

### M-004 — Relationship History

Relationship state must retain the events and interpretations that produced current trust, fear, obligation, resentment, and attachment.

**Evidence:** CASE-0001, CASE-0003, CASE-0004  
**Related hypotheses:** H-004, H-008  
**Confidence:** Moderate

### M-005 — Credible Forgetting

Forgetting must occur through explicit decay, interference, trauma, deception, suppression, or loss rather than implementation convenience.

**Evidence:** Ontological requirement; broader cross-medium testing needed  
**Confidence:** Low

### M-006 — Retrieval by Relevance

Memory retrieval must consider causal, relational, emotional, institutional, and thematic relevance rather than recency alone.

**Evidence:** Pilot synthesis  
**Related hypotheses:** H-008, H-014  
**Confidence:** Low

## Non-Goals

The Memory Engine is not a transcript store and should not expose all history equally at every moment.

## Validation Questions

- Can current state be explained by retained history?
- Do agents remember different versions of the same event?
- Are unresolved promises and injuries still available to later choices?
- Has forgetting been caused or merely assumed?
- Can institutions remember beyond individual membership?

## Open Questions

- What retention policies preserve meaning at scale?
- How should emotional salience interact with factual accuracy?
- How should contradictory testimony alter confidence in public memory?
